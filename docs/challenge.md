### Critical notebook findings

1. **Inverted rate calculation (critical)**
   - In `get_rate_from_column`, the rate is computed as `total / delays`.
   - The correct delay rate per category is `delays / total`.
   - Impact: categories with higher delays appear to have lower rates, and vice versa.

2. **Misleading scale label**
   - The column is named `Tasa (%)`, but values are not multiplied by `100`.
   - Impact: charts and narrative can be interpreted incorrectly.

3. **Boundary bug in `period_day`**
   - Strict comparisons (`>` and `<`) are used instead of inclusive boundaries.
   - Exact times such as `05:00`, `12:00`, and `19:00` are excluded from all buckets.
   - Evidence: `1230` null values were found in `period_day`.

4. **Pipeline inconsistency**
   - `training_data = shuffle(...)` is created but not used to build
     `features` and `target`.
   - Impact: indicates an incomplete or residual experimentation flow.

5. **Split without `stratify` in an imbalanced problem**
   - `train_test_split(..., random_state=42)` is used without stratification.
   - It does not break execution, but reduces methodological robustness.

6. **Fragile evaluation setup**
   - Only one train/test split is used.
   - There is no cross-validation or variability analysis across runs.

### Model conclusion (critical analysis)

Under the objective of delay detection (maximize `recall_1`):

1. **Best model in this notebook**:
   - **XGBoost with top-10 features + class balancing**.

2. **Why it should be selected in this phase**:
   - It achieved the best `recall_1` and best `f1_1` among the 6 runs.
   - It sacrifices global `accuracy`, but substantially improves minority-class
     detection (delays).

3. **Important nuance**:
   - The gap vs balanced LogisticRegression is very small.
   - With a more robust validation setup (k-fold or multiple seeds),
     a statistical tie is possible.

### Selected model for operationalization

The productionized model was aligned with the notebook conclusion:

- **Model**: `XGBoostClassifier`
- **Feature space**: fixed top-10 one-hot features
- **Class imbalance handling**: `scale_pos_weight = n_y0 / n_y1`
- **Current implementation**: `challenge/model.py` (`DelayModel.fit`)
- **Training artifact**: `data/model.skops` (generated by `data/train_model.py`)
